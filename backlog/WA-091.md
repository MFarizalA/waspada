# WA-091 · Flow-chart (node-graph) view of the debate, alongside the chat

**Candidates:** Claude or frontend. High **demo value** for the Agent-Society track — the
node graph makes the "six-agent society + bounded debate" legible at a glance, next to the
turn-by-turn transcript.

## Why
`AgentDialogue.tsx` renders the debate as a **linear chat transcript** (Round 1 challenge →
Round 2 rebuttal → Round 3 ruling → resolution). That's great for *reading* an argument, but it
doesn't show the **topology**: which agents exist, how the deterministic spine flows, and how each
dispute branches to a terminal state. The reference mock (a node canvas: `Start` → labeled edges
`Asks for help` / `Ask…` → `Conversation` / `Tool` nodes → `Tool Success`) is exactly the missing
view. We add it **beside** the chat, not instead of it — chat = the words, flow = the map.

## Design
- **Canvas beside the chat.** Split/side-by-side layout in the debate panel: transcript on one
  side, an interactive node-graph on the other (or a toggle on narrow screens).
- **Two layers of graph:**
  1. **Society spine (static topology):** `Data Engineer → Data Analyst → Actuary (risk_model) →
     Skeptic (risk_auditor) → Arbiter → Approval Gate`. The deterministic orchestrator flow,
     always shown so viewers see the whole society.
  2. **Per-dispute sub-flow (dynamic):** for a selected dispute from `agent_dialogue`, render
     `Skeptic challenge → Actuary rebuttal → Arbiter ruling → resolution`, with **labeled edges**
     mirroring the mock: `opens dispute`, `uphold` / `concede` (Round 2), `uphold` / `override` /
     `escalate` (Round 3), terminating in `upheld` / `overridden` / `escalated` badges.
- **Node content = real data.** Each node card shows the speaker label + a one-line claim (from
  `DisputeRound.claim`) + confidence, using the same `speakerColor` / i18n `speaker.*` labels the
  chat already uses. No new backend — reads `DashboardPayload.agent_dialogue` (`DisputeRecord[]`,
  each with `rounds: DisputeRound[]`, `resolution`, `resolved_by`).
- **Live mode.** Subscribe to the existing `useLiveDebateStream` SSE (`round` / `resolution`
  events): nodes/edges **light up as rounds arrive** — the "debating" spinner becomes the graph
  animating, the demo money-shot. No new endpoint.
- **Visual style = the mock, in our tokens.** Dotted-grid canvas, rounded node cards with a tinted
  header bar + icon, curved connectors, pill-shaped edge labels — but themed with the **cloud-blue
  design tokens** (`tokens.css`), not raw hex. Node header uses `--surface-tint`/`--cn-red-*`
  (brand), edges `--border-strong`, success/terminal states reuse the risk/resolution colors.
- **Library:** recommend **React Flow (`@xyflow/react`)** — it matches the mock's look (draggable
  nodes, curved edges, edge labels, minimap/zoom) with little custom code. Alternative: a
  hand-rolled SVG for the fixed spine if we want zero new deps / smaller bundle. Decide in
  implementation; either way keep it lazy-loaded so it doesn't bloat first paint.

## Acceptance
- The debate panel shows a node-graph beside the transcript: the static society spine **plus** the
  selected dispute's challenge→rebuttal→ruling→resolution branch with labeled edges.
- Nodes render real `agent_dialogue` data (speaker, claim, confidence) with localized labels; edge
  labels reflect the actual Round-2/Round-3 verdicts.
- In "Watch live" mode, nodes/edges appear as SSE `round`/`resolution` events arrive.
- Themed via `tokens.css` (re-themes with the palette; no hardcoded colors), responsive (graph
  scrolls/zooms inside its own container; page body never scrolls horizontally), a11y-labeled.
- Offline/empty state is graceful (no `agent_dialogue` → show just the static spine, no crash).

## Out of scope
- Editing the flow (this is a **read-only visualization**, not a builder like the mock's product).
- Backend changes — no new fields; reuse `agent_dialogue` + the existing SSE stream.
- Re-laying-out the whole dashboard; this lands inside the existing debate panel.
