# WA-039 · Dashboard Origination surface

## Goal

Render the Origination lane: application work-list with approve/refer/reject
decisions, origination health, and the same Agent Society debate panel (which
needs no change — `agent_dialogue` is lane-agnostic).

## What's hardwired today (must extend)

- `recommended_action` is a **hard TS union** `"call"|"watch"|"auto-cure"`
  (`dashboard/src/types.ts:33`).
- `ActionBadge` `STYLES` map keys exactly those three actions
  (`ActionBadge.tsx:6-8`) — an unknown action (e.g. `approve`) indexes
  `undefined` and crashes the badge.
- `PortfolioHealth.tsx` renders the collections NPL panel only.

## Files / interfaces

- `types.ts`: widen the action union per lane (`"approve"|"refer"|"reject"`),
  add `OriginationHealth` type + optional `priced_rate` on the account row.
- `ActionBadge.tsx`: styles for the three origination actions (+ new CSS action
  tokens in `tokens.css`; keep the WA-031 imperial-red/gold system — e.g.
  approve=jade, refer=gold, reject=red).
- A lane switch (or lane-tagged payload) in `App.tsx`; `PortfolioHealth`
  variant for approval-rate / band-mix / projected-default health.
- **i18n (WA-031)**: add EN/中文 keys for every new label (`action.approve` 批准,
  `action.refer` 人工复核, `action.reject` 拒绝, origination health labels).
- `AgentDialogue` panel: none (contract unchanged).

## Acceptance

- A `DashboardPayload` from the origination pipeline renders: work-list with
  the new badges, origination health panel, debate panel, drawer.
- Collections view pixel-unchanged; `tsc -b` + `vite build` green; both locales
  covered.

## Depends on

WA-037 (payload shape), WA-031 (i18n system — shipped).
