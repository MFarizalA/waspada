# WA-020 · QA the debate paths + secrets sweep (pre-public)

## Goal

Two deliverables before the repo goes public:

1. **Debate path tests** — scripted-`MockLLM` tests covering every dispute
   resolution path: open/concede/uphold/escalate, plus parse-degradation cases
   (unparsable LLM reply at each round → safe fallback). The debate is the
   track's namesake; every path must be proven, not assumed.

2. **Secrets sweep** — scan the repo before it goes public for any committed
   secret: `.env`, `secrets/`, deck files, API keys, AccessKeys, tokens. A
   leaked DASHSCOPE_API_KEY or OSS AccessKey on a public repo is a submission
   disaster and a real security incident.

## Files / interfaces

- `tests/qa/test_debate_paths.py` (NEW) — the scripted-MockLLM dispute tests.
- `tests/qa/test_debate_degradation.py` (NEW) — parse-failure-at-each-round cases.
- Secrets sweep: a scan (ripgrep for key patterns + manual check of `.env*`,
  `secrets/`, deck/slide files). Report findings; do not fix
  in place — escalate to Stefanie.

## Acceptance

- Every dispute resolution path has a passing test:
  - open dispute (Round 1 challenge succeeds)
  - concede (Round 2 → overridden, resolved_by=risk_model)
  - uphold + arbiter rules (Round 3 → upheld or overridden)
  - escalate (arbiter low-confidence OR auto-escalate → human gate →
    escalated_approved / escalated_rejected)
- Parse-degradation at each round (R1 unparsable → no dispute; R2 unparsable →
  auto-escalate; R3 unparsable → escalate) all pass.
- Secrets sweep report committed (or summarized to Stefanie); zero secrets in
  the repo. Confirm `.gitignore` covers `.env`, `secrets/`, `*.pkl`, `models/`,
  `data/` dumps.
- All existing tests stay green.

## Out of scope

- Live Qwen integration testing (needs creds — WA-018).
