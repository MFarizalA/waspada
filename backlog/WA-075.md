## Goal

A scenario for a deployment-focused video: prove the backend genuinely runs
on Alibaba Cloud, end to end, nothing faked.

## Scenario spine (draft)

1. **The IaC story** — `deploy/iac/main.tf` on screen: OSS three buckets, ACR,
   FC custom container, SLS audit, RDS MySQL, RAM least-privilege roles. "One
   `tofu apply` provisions all of it."
2. **The registry** — ACR console (or `imagetools inspect` output): the api
   image, linux/amd64.
3. **The function** — FC console: the function config (custom container,
   2048MB, 180s, VPC), trigger URL.
4. **Live proof** — `curl /api/health` → `{"status":"ok"}`; a real
   `POST /api/run?brain=mock` returning a full debate payload; RDS connection
   evidenced by login working (auth lives in MySQL).
5. **The domain story** — app.waspada.xyz once WA-067 lands (or the honest
   fcapp.run note if shot before then).
6. **Data plane** — OSS console: Raw/Staging/Mart buckets; note the DuckDB
   analytics path.
