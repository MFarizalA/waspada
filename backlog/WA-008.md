# WA-008 · Agent framework (shared substrate)

## Goal
The shared multi-agent substrate the orchestrator + pipeline agents build on: a
base agent interface, a step/tool-call protocol, structured run logging, and —
critically — the **human-in-loop approval hook** ("humans in control" rubric
requirement). Lane-agnostic; both lanes reuse it.

## Files / interfaces
- `waspada/agents/base.py`:
  - `Agent` base — `name`, `role`, `run(context) -> AgentResult`, a `tools`
    registry, structured step log.
  - `ApprovalGate` — `request(action, rationale) -> Approved|Rejected` — the
    human checkpoint; configurable auto-approve for non-prod runs (logged
    distinctly).
- `waspada/agents/protocol.py` — `AgentContext` (carries lane, data handles,
  prior results), `AgentResult` (status, artifact ref, notes), a simple
  handoff envelope.
- `waspada/agents/llm.py` — thin Gemini wrapper (free tier) for agent
  reasoning; **mockable** so the framework runs offline.
- `tests/test_framework.py` — a fake two-agent handoff + approval gate (approve
  + reject paths).

## Acceptance
- Two stub agents hand off via `AgentContext` and produce a combined result;
  logged.
- `ApprovalGate` blocks on rejection, proceeds on approval; auto-approve logged
  distinctly.
- LLM wrapper is mockable; framework runs end-to-end with a mock brain (no
  network).
- Base protocol documented; WA-009/WA-010 build on these names.

## Out of scope
Specialized pipeline agents (WA-009), orchestrator (WA-010), real LLM prompts
beyond stubs, frontend.

## Depends on
WA-001 (package, config).
