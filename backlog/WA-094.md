# WA-094 · Classical ML (PD) model — calibration, train/serve decoupling, upgrade path

**Candidates:** Claude or data/model lane. Improves the **model itself** — orthogonal to *where it
runs* (settled in `pd-model-hosting.md`: in-process, not PAI-EAS) and *how it's versioned* (WA-082).

## Why (grounded in the code)
`waspada/model/risk.py` trains a `LogisticRegression(class_weight="balanced")`, **fit per-run**, and
uses `predict_proba` directly as `p_default`. Three quality gaps:

1. **`p_default` is not calibrated.** `class_weight="balanced"` reweights the classes, which
   **distorts the predicted probabilities** — yet `p_default` is consumed as a *real* PD downstream
   (`expected_loss`, and the absolute bands WA-051/WA-093 want). An uncalibrated PD makes expected
   loss and absolute thresholds misleading.
2. **Fit-per-run instability.** Refitting on each incoming batch means a small/skewed batch (or the
   single-vintage shuffle-split fallback) produces a jumpy model — decisions wobble run to run for
   reasons unrelated to the borrower.
3. **Linear ceiling.** A single logistic model caps achievable AUC; a well-regularized GBM often
   lifts discrimination materially on this kind of tabular credit data — *if* we preserve the
   explainability the debate depends on.

## Design
- **Probability calibration.** Wrap the classifier in `CalibratedClassifierCV` (isotonic or sigmoid)
  on the hold-out split — or drop `class_weight` and correct the prior/threshold — so `p_default` is
  a **true probability**. Report a calibration metric (Brier score / reliability curve) alongside AUC.
- **Decouple train from serve.** Train on the **full historical dataset** → a versioned artifact
  (ties **WA-082**) → *score* with the frozen model; refit on a schedule/threshold, not every
  request. Removes fit-per-run instability and makes runs reproducible. Keep the current in-process
  re-fit as an offline fallback when no artifact is present.
- **Upgrade path (optional, behind a flag).** Add `HistGradientBoostingClassifier` (stdlib sklearn,
  no new heavy dep) as an alternative estimator with **monotonic constraints** where credit sense
  demands, and a **SHAP/contribution-based `explain()`** so the Risk Auditor still cites real
  drivers. Keep `LogisticRegression` as the interpretable baseline; choose by config, compare AUC +
  calibration before switching the default.
- **Preserve `explain()` + `band_edges`.** Whatever estimator wins, the signed-contribution drivers
  and band edges the Skeptic/Actuary cite in the debate must remain — the debate's evidence depends
  on them.

## Acceptance
- `p_default` is calibrated (Brier/reliability improves vs today); expected_loss + absolute bands use
  a true PD.
- Train/serve is decoupled: a run can score with a frozen versioned model and record *which* one
  (with WA-082); in-process re-fit remains as the offline fallback.
- Any estimator swap keeps `explain()` producing driver citations and AUC ≥ the current baseline.
- Offline/tests unchanged when no model artifact is configured.

## Out of scope
- **Where** the model is hosted — settled (`pd-model-hosting.md`: in-process, not EAS).
- **Versioning/persistence** plumbing — **WA-082** (this ticket *consumes* it for train/serve split).
- **Monitoring/drift** — **WA-093** (this ticket *produces* the calibrated scores it monitors).
