# WA-086 · Wire StdioClient into the live qwen debate (conformance A5-code)

**Candidates:** Claude or backend. Closes the *code* half of gap A5 from
`backlog/architecture-conformance-review.md` (the *doc* half was fixed in
`feature/conformance-fixes` `a086998`). **Optional rubric-strengthener, not a divergence
fix** — the doc is already honest.

## Why
The rubric's headline is a **real MCP integration**. WASPADA has a genuine stdio MCP
server (`waspada/mcp/server.py`, `python -m waspada.mcp.server`) and a real `StdioClient`
(`mcp/client.py`) — but the **live pipeline wires `InProcessClient`** (`orchestrator.py`
`_wire_analyst_aggregates`), and only when the Data Analyst emitted aggregates; otherwise
the auditor falls back to local in-memory stubs. The genuine stdio protocol round-trip runs
**only in a verify script** (`scripts/wa069-verify-mcp.py`), not any user-facing run. So the
"agents consume MCP over stdio" story is demonstrated out-of-band, not live.

## Goal
Make the stdio MCP round-trip real in the **`brain=qwen`** live path, so the MCP claim holds
under a judge's poke — while keeping the pipeline robust.

## Approach
- In the `brain=qwen` path, attach a **`StdioClient`** (spawn `python -m waspada.mcp.server`)
  to the Risk Auditor instead of / in addition to `InProcessClient`.
- **Fallback:** if the subprocess can't spawn (or errors), fall back to `InProcessClient` —
  never fail the run for a transport choice.
- Gate behind config (e.g. `WASPADA_MCP_TRANSPORT=stdio`) so mock/offline stays InProcess.

## The real risk (why this is a ticket, not a same-session fix)
**FC-sandbox subprocess execution is unverified** — HACKATHON.md:558 explicitly flags
"subprocess for stdio MCP inside FC's sandbox unverified." Function Compute custom containers
*can* spawn subprocesses, but this needs a **live FC smoke test** before trusting it in the
demo. A stdio path that hangs or can't spawn inside FC would be worse than the honest
InProcess default.

## Acceptance
- A `brain=qwen` run cites auditor evidence via a **real stdio MCP round-trip**, verified
  **on deployed FC** (not just locally).
- Clean fallback to `InProcessClient` on any stdio failure (run never breaks).
- Offline/tests unchanged (InProcess remains the default).

## Out of scope
- The doc honesty fix (done, `a086998`).
- Replacing InProcess for the mock/offline path (keep it — it's the reliable default).

## Alternative (if FC stdio proves flaky)
Keep InProcess live + **feature the stdio round-trip in the deployment video / verify script**
(the doc is already honest about this). Lower risk; still demonstrates real MCP.
