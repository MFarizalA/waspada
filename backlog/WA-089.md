# WA-089 · Pluggable data-source layer (LC / synthetic / Bondora / DB → RawLoans)

**Candidates:** Claude or backend/data. Makes the pipeline **source-agnostic**: the same
downstream (features → score → debate) runs on any source, each mapping to the frozen `RawLoans`
contract.

## Why
The source is currently hard-wired to the Lending Club CSV (WA-078 loader). To (a) swap in a
**legal-clean synthetic** source on demand, (b) offer alternative real datasets (Bondora), and
(c) be ready for a real **SQL/core-banking source** later, the ingest needs a **selectable source
layer** — while everything past `RawLoans` stays unchanged.

## Design
- A `Source` abstraction (or dlt sources) selected by **`WASPADA_DATA_SOURCE`**:
  - `lending_club` (default) — CSV → RawLoans (the existing WA-078 loader).
  - `synthetic` — generate RawLoans directly (built-in generator; optional **SDV** trained on the
    CC0 LC data for realism). **No license/PII — freely publishable.**
  - `bondora` — public EU P2P set → RawLoans (schema remap).
  - `sql` (future) — dlt SQL source + incremental cursor/CDC from a core-banking/LMS DB.
- Each source yields a **`RawLoans`-conformant Arrow table**; `validate_table(RawLoans)` is the shared gate.
- Downstream is untouched (it only ever sees `RawLoans`).

## Acceptance
- Changing `WASPADA_DATA_SOURCE` swaps the source; the pipeline runs end-to-end unchanged on each.
- `synthetic` produces a schema-valid, publishable RawLoans set with zero real records.
- `lending_club` remains the default; offline/tests unchanged.
- Tests: each source → `validate_table(RawLoans)` passes; the selector resolves correctly.

## Out of scope
- The OSS landing/partitioning (WA-078/047/088) — this ticket produces the RawLoans table; landing is separate.
- Real DB connection creds/CDC tuning (the `sql` source is scaffolded, not productionized here).

## Notes
Pairs with the data-pipeline architecture (`backlog/data-pipeline-architecture.md`). The `synthetic`
source is the legal-clean escape hatch (see the data-source research). Touches the ingest/loader
layer — coordinate with the data lane.
