# WA-036 · Model parameterization for the Origination lane

## Goal

Make `waspada/model/risk.py` serve both lanes. The sklearn pipeline is
**structurally lane-agnostic but feature-hardwired**: `NUMERIC_FEATURES`/
`CATEGORICAL_FEATURES`/`FEATURE_COLUMNS` name collections features
(`payment_ratio`, `outstanding_ratio`, `loan_age` — `risk.py:72-77`), and the
**vintage split reconstructs issue year from `loan_age` + `as_of`**
(`issue_year_from_frame` `risk.py:95-111`, `_vintage_split` `:114-170`) — an
application has no `loan_age`.

## Files / interfaces

- Parameterize the feature lists: a per-lane feature spec (e.g. a small
  `LaneFeatures` mapping or explicit params on `train`/`predict`) — collections
  defaults unchanged; origination uses the WA-035 feature set.
- Replace the split for origination: cohort by **application_date** (train older
  application cohorts, test newer — the same out-of-time discipline, without the
  `loan_age` reconstruction). Keep the seeded-shuffle degenerate fallback.
- `LEAKAGE_EXCLUDED` per lane (`risk.py:81-86`).
- `predict()` carry-forward columns (`delinquency_status`/`label_default`,
  `:281-333`) are collections monitoring aids — make optional/lane-aware.
- `_quintile_bands` (`:252-278`) reuses as-is (relative Q1..Q5 on application PD).

## Acceptance

- Collections train/predict byte-identical with defaults (existing
  `tests/test_model.py` + `tests/qa/test_leakage.py` green untouched).
- Origination train/predict produces `ScoredApplications` with hold-out AUC >
  chance on the synthetic snapshot; split windows disjoint by application cohort.
