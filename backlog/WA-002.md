# WA-002 · BigQuery ingest layer

## Goal
The shared engine's data door: a reusable BigQuery client that reads creds from
env, runs a query, and returns Arrow (fast path) — plus a freshness/schema check
the Ingest agent (WA-009) will reuse. Collections reads the loans table; the same
client serves the Origination lane later.

## Files / interfaces
- `waspada/data/bq.py`:
  - `BigQueryClient` — built from `BQ_PROJECT` (env).
    - `query(sql) -> pyarrow.Table` (uses BigQueryStorage for fast Arrow export).
    - `table_meta(dataset_table) -> {n_rows, schema, freshness}`.
  - `fetch_loans(lane="collections") -> pa.Table` — returns a `RawLoans`-shaped
    Arrow table (WA-001 contract) for the configured lane/table.
- `.env.example` — add `GOOGLE_APPLICATION_CREDENTIALS`, `BQ_PROJECT`,
  `BQ_DATASET`, `BQ_TABLE`.
- `tests/test_bq_smoke.py` — skipped if no creds; when creds present, asserts
  `table_meta` returns non-zero `n_rows`.

## Acceptance
- With valid sandbox creds: `fetch_loans()` returns an Arrow table whose columns
  are a superset of `RawLoans` (smoke test on a small `LIMIT`).
- `table_meta()` reports row count + freshness.
- No creds → clear `RuntimeError("BQ credentials not configured")`, never a
  silent failure.
- Runs on the host (CPU Arrow); no GPU needed here.

## Out of scope
Loading data into BQ (WA-003), feature engineering (WA-004), anything GPU.

## Depends on
WA-001 (`RawLoans` schema, `config`).
