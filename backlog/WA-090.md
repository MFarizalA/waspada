# WA-090 · Activate the medallion — write Staging (Silver) + Mart (Gold)

**Candidates:** Claude or backend/data. Closes the "provisioned-but-dead buckets" gap the
architecture review (A-register B1) and infra audit both flagged.

## Why
The IaC provisions **three** medallion buckets (`raw`/`staging`/`mart`) and grants FC PutObject on
staging+mart — but **no code writes to staging or mart** (verified: zero `put_object` in `waspada/`).
So "three-bucket medallion" is one live tier (Raw) wearing a three-tier costume. This ticket makes
the upper tiers real: the Data Analyst's derived features land in **Silver**, the served payload lands
in **Gold**.

## Design
- **Silver (Staging):** the Data Analyst writes its `FeatureFrame` + aggregates to
  `oss://waspada-prod-staging/features/dt=<batch>/…` (parquet). Preferred via **dlt filesystem
  destination** (WA-083 Option B) for lineage; or a direct oss2 put via the shared write helper.
- **Gold (Mart):** Insight writes the assembled `DashboardPayload` (+ `ScoredAccounts`) to
  `oss://waspada-prod-mart/payload/dt=<batch>/…`. The dashboard can then read a **pre-computed Gold
  payload instantly** (fast cold path) instead of always running the pipeline live.
- Writes are **additive + partitioned** (never mutate); a failed write leaves the prior tier intact.

## Acceptance
- A run writes FeatureFrame/aggregates to Staging and the DashboardPayload to Mart, both
  partitioned + schema-checked.
- The dashboard (or `/api/run?cached=1`) can serve the latest Gold payload without a live pipeline run.
- Offline/tests unchanged when OSS isn't configured (writes are best-effort/guarded, like the SLS sink).

## Out of scope
- The OSS **write path helper** itself — build it once (shared with WA-082 model binary, WA-083 dlt,
  WA-087 secrets); this ticket consumes it.
- Pushdown reads (WA-047 §read-path).

## Notes
Activating Silver/Gold also makes the batch-refresh story (WA-088) complete: the scheduled producer
can pre-compute + land the Gold payload so the demo dashboard is instant and always-fresh. Data lane.
