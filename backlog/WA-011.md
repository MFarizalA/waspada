# WA-011 · EWS dashboard (Collections analyst view)

## Goal
The analyst-facing Collections/EWS view: ranked work-list, per-account risk
scores, portfolio health (NPL ratio, roll rates, vintage by product/region/
branch), and segment alerts. Reads the **frozen `DashboardPayload` JSON**
(contract locked in WA-001). This is what the analyst actually uses.

## Files / interfaces
- `dashboard/` — React + TypeScript (Vite). Reads `DashboardPayload` JSON
  (served or static for the demo).
  - `WorkList` table — sortable by `p_default`, action badges, top-N.
  - `AccountDrawer` — per-account score, band, segment, recommended action.
  - `PortfolioHealth` — NPL ratio, roll-rate by cohort, vintage chart.
  - `Alerts` — segment deterioration alerts with severity.
- Contract source: `waspada/schema.py` → `DashboardPayload` (frozen). Mirror as
  a TS type in `dashboard/src/types.ts`.

## Acceptance
- Renders a `DashboardPayload` sample (committed fixture) without errors.
- Work-list sorts by score; portfolio charts show NPL/roll-rate/vintage; alerts
  render with severity.
- `npm run dev` loads; `npm run build` is clean.
- No live backend required for the demo (fixture-driven); a real-run mode
  points at the orchestrator output path.

## Out of scope
Origination views (deferred), agent internals, model training, the benchmark,
auth/multi-tenancy.

## Depends on
WA-001 (`DashboardPayload` — **frozen**, so frontend can build in parallel with
backend once WA-001 lands).

## Note
Contract is frozen on the dashboard payload — Kirana can start from a committed
fixture JSON as soon as WA-001 is in, without waiting on the pipeline.
