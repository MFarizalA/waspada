# WA-050 · Let the Actuary introspect its own model (grounded defense, not confabulation)

## The bug

**The Actuary defends a model it cannot see.**

`_rebuttal_prompt()` (`risk_model.py:176-201`) passes the defense exactly four
things: the band, `p_default`, the Skeptic's challenge, and **raw feature values**
(`_feature_facts`, `risk_model.py:270-284`).

It does **not** pass the model's coefficients or per-feature contributions. Grep
for `coef_|feature_names_out|named_steps` across the whole tree: **zero matches.**

So the LLM "defending the score" has **no access to why the model produced it**.
It generates a plausible-sounding *post-hoc rationalization* from the same numbers
the Skeptic already has. HACKATHON.md's framing — *"a number can't argue for
itself, so an LLM is assigned as its defense counsel"* — is elegant, but **the
counsel has never met its client**, and its `concede` / `uphold` verdict is
therefore not grounded in the model's actual reasoning.

## The fix

New `explain(model, features, loan_id) -> List[Tuple[str, float]]` in
`waspada/model/risk.py`:

- `pipeline.named_steps["clf"].coef_` — the fitted LogisticRegression weights
  (`risk.py:185-186`).
- `pipeline.named_steps["pre"].get_feature_names_out()` — post-`ColumnTransformer`
  names (StandardScaler numerics + one-hot categoricals, `risk.py:178-184`).
- Transform the account's row, multiply, return the **top-N signed logit
  contributions**.

~25 lines, **no new dependency** — and entirely net-new surface, since nothing in
the repo touches `coef_` today.

Feed it into `_rebuttal_prompt()` so the defense reads:

> Your model's drivers for this account: `rate=24.0 (+0.81 logit)`,
> `dti=30.0 (+0.44)`, `payment_ratio=0.95 (−0.32)`.

Give the **Skeptic** the same block in its challenge prompt (`risk_auditor.py:468`)
— an evidence-grounded challenge beats a vibes-based one, and it makes the "every
claim cites evidence" rule (HACKATHON.md § debate protocol) actually *enforceable*
rather than merely asserted.

## Why it matters beyond the debate

This is exactly the **feature-attribution surface the dashboard needs** to answer
*"why is this account High?"* — the single most credible thing you can put in front
of a collections lead, and the thing a regulator asks for first. The debate fix and
the explainability feature are the same ~25 lines.

## Acceptance

- `explain()` returns signed contributions summing (with the intercept) to the
  logit of the model's own `p_default` — assert round-trip within tolerance.
- The rebuttal prompt contains the drivers; a test asserts the Actuary's evidence
  cites at least one contribution term.
- Works on the offline path (MockLLM) — `explain()` is pure sklearn, no network.
- Offline suite green.
