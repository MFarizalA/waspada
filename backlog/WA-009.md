# WA-009 · Pipeline agents (ingest / analytics / risk-model / insight)

## Goal
The four specialized agents the judges see — thin wrappers over the pipeline
components (WA-002/004/005/006), each with a clear role, that the orchestrator
(WA-010) coordinates. Each runs its stage, checks results, reports structured
findings. Ingest → Analytics → Risk-Model → Insight.

## Files / interfaces
- `waspada/agents/ingest.py` — wraps `waspada.data.bq`; freshness/schema
  checks, returns `RawLoans` handle, flags staleness.
- `waspada/agents/analytics.py` — wraps `waspada.features.collections`; runs
  `build_features`+`build_label` via WSL, returns `FeatureFrame`.
- `waspada/agents/risk_model.py` — wraps `waspada.model.risk`; trains/predicts,
  returns `ScoredAccounts`, flags low-confidence bands.
- `waspada/agents/insight.py` — wraps `waspada.insight.ranking`; builds
  work-list + portfolio health + alerts, returns `DashboardPayload`; composes a
  plain-language alert summary.
- Each extends `waspada.agents.base.Agent`; uses `ApprovalGate` where a human
  threshold applies (insight agent surfaces the work-list for approval).

## Acceptance
- Each agent produces the contract artifact for its stage on a small
  end-to-end run.
- Ingest agent raises on stale/missing data; analytics agent surfaces
  feature-null rates; risk-model agent flags score bands.
- Insight agent emits ≥1 human-readable alert string alongside the payload.
- Runs offline with the mock LLM; no network for the integration test.

## Out of scope
The orchestrator (WA-010), dashboard UI (WA-011), Origination agents (deferred),
new pipeline logic (these wrap existing components).

## Depends on
WA-008 (framework), WA-002/004/005/006 (components).

## Note
Bundled as one ticket (not four) deliberately — these are thin wrappers; four
separate dispatches would burn Z.ai throttle budget. If the brief proves too
wide, split insight out (it has the most logic) and keep the other three.
