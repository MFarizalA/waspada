# WA-004 · Collections feature + label engineering (cuDF, GPU)

## Goal
The GPU feature step for the Collections lane — the analytics core. The source
data is **cross-sectional** (one row per loan — a snapshot), so this builds a
**cross-sectional** feature set and an **eventual charge-off / default** label,
NOT month-over-month roll rates or a 30-day roll label (those aren't computable
from a snapshot). Runs on GPU via cuDF in WSL; emits a `FeatureFrame` per the
frozen contract.

## Files / interfaces
- `waspada/features/collections.py`:
  - `build_features(raw: pa.Table) -> FeatureFrame` — invoked via
    `waspada.wsl.run_gpu`. Snapshot + behavioral features:
    - Snapshot: `amount, term, rate, grade, annual_income, dti, purpose,
      region`.
    - Behavioral (derived): `loan_age` (months from `issue_date` to as-of),
      `payment_ratio` = total_paid/amount, `outstanding_ratio` =
      outstanding_principal/amount, current delinquency status / bucket.
  - `build_label(raw) -> Series[bool]` — `label_default` = **eventual charge-off
    / default**, derived from final `current_status` (e.g.
    `loan_status ∈ {Charged Off, Default}` → True). As-of date configurable so
    outcome-derived fields never leak into training features.
  - Portfolio-level roll rates are NOT computed here (need a panel — stretch);
    they live in WA-006 as aggregates only, if computed at all.
- `gpu/run_features.py` — WSL-side entry script `run_gpu` invokes: reads
  parquet in, writes `FeatureFrame` parquet out.
- `tests/test_features.py` — tiny synthetic frame; asserts label correctness on
  a hand-built case (one Charged Off → True, one Current → False) + feature
  dtypes + no nulls in required fields.

## Acceptance
- `build_features()` on a 10k-row sample returns a `FeatureFrame` with all
  contract features populated; no nulls in required fields.
- The hand-built label test passes (Charged Off/Default → True; Current/Fully
  Paid → False).
- Runs via WSL (`wsl -e /root/rapids/bin/python gpu/run_features.py …`); host
  never imports cuDF.
- VRAM-safe (4 GB cap): processes in slices + concatenates if needed.
- **No per-account roll rates, no 30-day roll label** — cross-sectional only.

## Out of scope
The model (WA-005), ranking (WA-006), CPU baseline (WA-007), ingestion itself,
portfolio-level roll-rate aggregates (WA-006).
