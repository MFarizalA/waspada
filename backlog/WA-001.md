# WA-001 ¬∑ Repo scaffold + frozen data contract

## Goal
Stand up the shared WASPADA Python package every downstream ticket builds
against: package layout, config/env loading, the WSL‚ÜíGPU entrypoint helper, and
the **frozen data-contract schemas** (raw ‚Üí features ‚Üí scores ‚Üí dashboard
payload). This is the shared spine ‚Äî nothing else builds until the contract is
locked here.

## Files / interfaces
- `waspada/__init__.py`, `waspada/config.py` ‚Äî env/config load (`.env`:
  `BQ_PROJECT`, `BQ_DATASET`, `BQ_TABLE`, `WASPADA_LANE`); lane switch
  (collections | origination).
- `waspada/schema.py` ‚Äî **the frozen contract** (dataclasses / TypedDict). NB:
  the source data is **cross-sectional** ‚Äî one row per loan (a snapshot of
  current loan_status + aggregate payment totals), NOT a monthly payment panel.
  So there are no per-account month-over-month roll rates and no computable
  30-day roll-to-NPL label. The label is **eventual charge-off / default**
  (derived from final status); roll rates are a **portfolio-level aggregate**
  only, and true roll rates need a panel (Freddie Mac ‚Äî stretch).
  - `RawLoans` ‚Äî raw rows (one per loan): `loan_id, amount, term, rate, grade,
    annual_income, dti, issue_date, purpose, region, outstanding_principal,
    total_paid, current_status` (mapped from LendingClub columns in WA-003).
  - `FeatureFrame` ‚Äî per-loan: `loan_id, features‚Ä¶` (snapshot + behavioral:
    loan_age, payment_ratio = total_paid/amount, outstanding/amount, current
    delinquency status, dti, grade, term), `label_default`, `as_of_date`.
  - `label_default` ‚Äî bool: **eventual charge-off / default** (from final
    `current_status`), NOT a 30-day roll.
  - `ScoredAccounts` ‚Äî `loan_id, p_default` (P eventual default), `score_band,
    segment{product,region}, recommended_action`.
  - `DashboardPayload` (JSON) ‚Äî `{work_list:[ScoredAccounts], portfolio_health:
    {npl_ratio, vintage_default_rate, status_mix}, alerts:[]}`. (True roll rates
    deferred to the Freddie Mac panel stretch ‚Äî not in the MVP payload.)
- `waspada/wsl.py` ‚Äî `run_gpu(args)` helper: shells out to
  `wsl -e /root/rapids/bin/python <script> <args>`, captures stdout/stderr,
  surfaces non-zero exit. The single chokepoint for all GPU/data steps.
- `.env.example` ‚Äî documents every var; `.env` stays gitignored.
- `tests/test_wsl_smoke.py` ‚Äî asserts `run_gpu(["-c","import cudf, cuml;
  print('ok')"])` returns `ok`.

## Acceptance
- `python -c "import waspada"` works from repo root.
- `waspada.wsl.run_gpu(["-c","import cudf, cuml; print('ok')"])` prints `ok`
  (proves the Python‚ÜíWSL‚ÜíRAPIDS bridge).
- `waspada.schema` exports the four contract types; a unit test round-trips a
  `FeatureFrame` ‚Üí parquet ‚Üí back.
- Contract is **frozen and documented** (docstrings per type). Downstream
  tickets cite these names verbatim.

## Out of scope

(WA-004), frontend.
