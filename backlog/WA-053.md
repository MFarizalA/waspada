# WA-053 · The audit stream can't answer the questions it exists for

## The bug

The SLS audit record carries `run_id / agent / action / model / tokens / latency /
resolution` — but **`tokens` and `latency` are always `None`**. Honestly so: nothing
in the pipeline measures them.

So the "queryable audit stream" cannot answer:

- *"What did this run cost?"* — the question that makes it valuable to a lender.
- *"Where is the latency?"* — the question that makes it actionable to us.

And the **efficiency claim rests on exactly these numbers.** Today the *benchmark
harness* measures cost, and the *audit trail* doesn't — which means "run 2 spends
measurably fewer LLM calls" (HACKATHON.md:476-477) is demonstrable in a bench script
but **not observable in the deployed product**.

## The fix

- Capture token usage from `LLM.chat()` / `LLM.complete()` (the Qwen response
  already carries a `usage` block; MockLLM returns zeros).
- Capture elapsed-ms in `Agent.step()`.
- Populate the two fields that already exist in the audit record shape.

Then the WA-026 second efficiency axis becomes measurable **in production**, not
only in `bench_society/`.

## Acceptance

- A real run's SLS/local audit records carry non-null `tokens` and `latency`.
- Offline (MockLLM) runs still emit records — with zero tokens, real latency.
- Run-1 vs run-2 token totals are queryable from the audit stream alone, with no
  bench harness.
- Offline suite green.
