# WA-079 · Prove the live debate fires on the REAL dataset

**Depends on:** WA-077 (API cutover) + WA-078 (real data in OSS).

## Goal

The synthetic demo set was engineered to produce disputes. The real Lending
Club dataset must also produce a visible, credible debate — the video money
shot (WA-074) needs it, and it must happen on REAL data now.

## What to do

1. Run the full pipeline (mock brain first, then `brain=qwen` K=1-2 with
   Stefanie's token sign-off) against the real dataset via the prod API.
2. Verify: at least one contested account produces the full protocol —
   challenge → defense (Actuary citing real fitted drivers, WA-050) →
   Arbiter ruling — with `agent_dialogue` populated in the payload.
3. If NOTHING disputes on the real data: diagnose why (band distribution on
   real data vs the WA-051 absolute edges? top-K selection skipping
   boundary accounts? gate thresholds?) and tune the SELECTION (which
   accounts surface to the debate), NOT the data and NOT the thresholds to
   force drama. The fix must be defensible: e.g. prefer accounts near band
   edges in the top-K debate selection — that's analytically honest.
4. Also verify the live-qwen run completes within the FC timeout (180s) on
   real data — the 120s timeout Bimo hit (WA-069) must be re-tested after
   cutover; if it still times out, report timing breakdown (where it dies).

## Acceptance

- Evidence (run output + payload excerpt) of a complete real-data debate:
  rounds, resolutions, dialogue, portfolio EL present.
- Live qwen run completes in prod within timeout, or a precise timing
  diagnosis of where it dies.
- `backlog/WA-079-report.md` with both.

## Out of scope

- Changing model coefficients, thresholds, or data to manufacture disputes.
- FC timeout bump (if needed, that's a one-line IaC change for Stefanie —
  report it, don't do it).
