## What shipped (merged, pushed)

### WA-048 · The debate now changes the work-list

Before this, a dispute where the model **conceded** still shipped the account
with its original band and its original `call` action. This was visible in our
own demo fixture: `LN00999998`'s rationale read *"…instead of a collector call"*
while the work-list row still said **call**.

- The society now writes `final_band` + `override_reason` back onto the scored
  book, and `recommended_action` follows it.
- **`p_default` / `score_band` are never rewritten** — the model's score stays
  the auditable statistical fact; the override is a documented, reason-coded
  layer on top (this also lets us *measure* society-vs-model divergence).
- **Asymmetric governance:** an escalation (society raises risk) auto-applies —
  worst case a wasted call; a de-escalation (society cancels a call) requires a
  human sign-off — worst case a missed default. Reuses the existing
  `escalated_approved` / `escalated_rejected` states.
- Dashboard shows the model→final band transition with the override reason
  (EN/中文). The `sample-payload.json` fixture was regenerated so the demo shows
  the loop closed.

### WA-049 · The audit can finally catch a false negative

The Skeptic only ever audited the top-K riskiest — accounts the model *already*
flagged — so it could only review candidate false **positives** (a wasted call:
cheap). The expensive error, a "Very Low" account that rolls to NPL, was never
sampled, and the symmetric half of the admissibility rule was unreachable dead
code.

- The audit slice is now **stratified** (`riskiest` / `boundary` /
  `contradictory`) at the **same K and the same ≤ K×3 call ceiling**.
- The `contradictory` stratum (low band + delinquent) is a zero-LLM-cost rule
  screen — the false-negative catcher. A planted false negative now reaches the
  audit and is disputed.
