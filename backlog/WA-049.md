# WA-049 · Fix the audit sample so false negatives are reachable

## The bug

**The Skeptic structurally cannot catch the model's most expensive error.**

The audit slice is **top-K by `p_default`** (`risk_auditor.py:199-201`) — the
accounts the model *already* flagged as riskiest. The code comment even owns the
choice: *"the riskiest slice is where a wrong band costs most."*

But look at the error economics of collections:

| Error | What happens | Cost |
|---|---|---|
| **False positive** — model says risky, isn't | a wasted collector call | cheap |
| **False negative** — model says safe, rolls to NPL | the loss the product exists to prevent | **expensive** |

Auditing only the top-K means **the Skeptic only ever reviews candidate false
positives.** It can downgrade; it can never upgrade.

The admissibility rule is *symmetric in principle* — `risk_auditor.py:520-526`
allows `Very Low` band + auditor `High` → dispute — but that case **can never
occur in production**, because low-band accounts are never sampled into the audit
set. Half the gate is dead code.

**The society is architecturally incapable of catching the error class that
motivates the product.**

## The fix — stratified draw, same K, same cost

Replace the top-K slice at `risk_auditor.py:199-201`:

```python
AUDIT_MIX = {"riskiest": 3, "boundary": 2, "contradictory": 3}   # sums to k=8
```

- **riskiest** — today's behaviour (top by `p_default`): catches over-calling.
- **boundary** — nearest a band edge: where the model is least certain.
- **contradictory** — **the false-negative catcher**: low band but adverse raw
  evidence (`delinquency_status ∈ {"Default","31-120","16-30"}` while
  `score_band ∈ {"Very Low","Low"}`). A pure rule-based screen — **zero extra LLM
  cost** — reusing the `_NPL_BUCKETS` set already defined at `ranking.py:72`.

**Cost is unchanged.** Still ≤ K accounts audited → the ≤ K×3 LLM-call ceiling
(HACKATHON.md § bounded debate) is untouched. This is strictly better coverage at
an identical budget.

Backfill rule: if a stratum is short (e.g. no contradictory accounts on a clean
book), spill its quota to `riskiest` so the audit always spends its full K.

## Acceptance

- A fixture with a **planted false negative** — a delinquent account the model
  bands `Very Low` — is drawn into the audit set and disputed. I.e.
  `_should_dispute("Very Low", "High")` becomes **reachable end-to-end**, which it
  is not today.
- The audit still spends ≤ K accounts (assert the call ceiling holds).
- A clean book with no contradictory accounts backfills to `riskiest` and behaves
  exactly as today (regression).
- Offline suite green.
