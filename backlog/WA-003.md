# WA-003 · LendingClub → BigQuery acquisition (owner-provided file)

## Goal
Acquire the dataset that unblocks the Collections/EWS hero lane: load an
**owner-provided LendingClub file** (local path or URL — the owner supplies it;
do NOT assume Kaggle credentials, Bimo won't have them) into the BigQuery
sandbox as the Collections source table, mapped to the `RawLoans` contract. One
dataset, two lanes: LC's `current_status` + payment totals → Collections (now);
LC's application attributes → Origination (later).

## Files / interfaces
- `waspada/data/load_lendingclub.py` — takes an owner-provided file path or URL
  (arg / env `LC_SOURCE_PATH`), stages the extract, maps raw LC columns to the
  `RawLoans` contract (`loan_id, amount, term, rate, grade, annual_income, dti,
  issue_date, purpose, region, outstanding_principal, total_paid,
  current_status`), and loads into `BQ_DATASET.BQ_TABLE`.
  - Likely LC→contract map: `id`→loan_id, `loan_amnt`→amount, `term`,
    `int_rate`→rate, `grade`/`sub_grade`→grade, `annual_inc`→annual_income,
    `dti`, `issue_d`→issue_date, `purpose`, `addr_state`→region,
    `out_prncp`→outstanding_principal, `total_pymnt`→total_paid,
    `loan_status`→current_status. Finalize in `lc_columns.md`; document coercion.
- `waspada/data/lc_columns.md` — raw LC field → contract field map (provenance).
- `scripts/load_data.sh` — thin wrapper: `python -m waspada.data.load_lendingclub
  "$LC_SOURCE_PATH"` against the sandbox project.

## Acceptance
- `BQ_DATASET.BQ_TABLE` exists; `SELECT COUNT(*)` returns the loaded row count.
- `LIMIT 10` returns rows whose columns are a superset of `RawLoans` (WA-001).
- Idempotent — re-run doesn't duplicate (`WRITE_TRUNCATE` / load-job overwrite).
- Records the sample size used (honesty for the benchmark later — WA-007).
- Reads from the owner-provided path/URL; **no Kaggle login anywhere**.

## Out of scope
Freddie Mac scale-up (stretch, not MVP), feature math, the BQ client itself
(WA-002).

## Depends on
WA-002 (client), WA-001 (schema).
