# WA-003 ¬∑ LendingClub ‚Üí BigQuery acquisition (owner-provided file)

## Goal
Acquire the dataset that unblocks the Collections/EWS hero lane: load an
**owner-provided LendingClub file** (local path or URL ‚Äî the owner supplies it;
do NOT assume Kaggle credentials, Bimo won't have them) into the BigQuery
sandbox as the Collections source table, mapped to the `RawLoans` contract. One
dataset, two lanes: LC's `current_status` + payment totals ‚Üí Collections (now);
LC's application attributes ‚Üí Origination (later).

## Files / interfaces
- `waspada/data/load_lendingclub.py` ‚Äî takes an owner-provided file path or URL
  (arg / env `LC_SOURCE_PATH`), stages the extract, maps raw LC columns to the
  `RawLoans` contract (`loan_id, amount, term, rate, grade, annual_income, dti,
  issue_date, purpose, region, outstanding_principal, total_paid,
  current_status`), and loads into `BQ_DATASET.BQ_TABLE`.
  - Likely LC‚Üícontract map: `id`‚Üíloan_id, `loan_amnt`‚Üíamount, `term`,
    `int_rate`‚Üírate, `grade`/`sub_grade`‚Üígrade, `annual_inc`‚Üíannual_income,
    `dti`, `issue_d`‚Üíissue_date, `purpose`, `addr_state`‚Üíregion,
    `out_prncp`‚Üíoutstanding_principal, `total_pymnt`‚Üítotal_paid,
    `loan_status`‚Üícurrent_status. Finalize in `lc_columns.md`; document coercion.
- `waspada/data/lc_columns.md` ‚Äî raw LC field ‚Üí contract field map (provenance).
- `scripts/load_data.sh` ‚Äî thin wrapper: `python -m waspada.data.load_lendingclub
  "$LC_SOURCE_PATH"` against the sandbox project.

## Acceptance
- `BQ_DATASET.BQ_TABLE` exists; `SELECT COUNT(*)` returns the loaded row count.
- `LIMIT 10` returns rows whose columns are a superset of `RawLoans` (WA-001).
- Idempotent ‚Äî re-run doesn't duplicate (`WRITE_TRUNCATE` / load-job overwrite).
- Records the sample size used (honesty for the benchmark later ‚Äî WA-007).
- Reads from the owner-provided path/URL; **no Kaggle login anywhere**.
