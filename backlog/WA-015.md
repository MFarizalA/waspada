# WA-015 · MCP server (portfolio_stats, lookup_account) + Skeptic integration

## Goal

Build a real Model Context Protocol server (`waspada/mcp/`) serving two tools —
`portfolio_stats` and `lookup_account` — over the data the analytics agent
already computed (DPD buckets, payment ratios, vintage cohorts, segment health).
The `RiskAuditorAgent` (built in WA-014) consumes these tools through an MCP
client session, so the Skeptic's cited evidence is pulled live from the actual
loan book, not a static corpus. This is the rubric's explicit MCP integration.

The Skeptic's function-calling loop decides WHEN to call a tool — that's the
Qwen native function-calling the rubric rewards. Hard-wired Python calls would
forfeit the point.

## Files / interfaces

- `waspada/mcp/server.py` (NEW) — MCP server exposing:
  - `portfolio_stats(segment?)` → NPL ratio, vintage default rate, segment
    health for a product×region slice (or portfolio-wide if no segment). Backed
    by the analytics agent's FeatureFrame aggregates.
  - `lookup_account(loan_id)` → the account's feature row (payment_ratio,
    dti, delinquency_status, loan_age, outstanding_principal, etc.) — exactly
    what the Skeptic cites as evidence.
- `waspada/mcp/client.py` (NEW) — thin MCP client session the Skeptic uses to
  call the server (stdio transport; SSE is stretch WA-021).
- `waspada/agents/risk_auditor.py` — replace the local stub tools (from WA-014)
  with MCP client calls. The Skeptic registers the MCP tools via
  `register_tool` (same pattern as ingest's `fetch` → stubbable in tests).
- `waspada/mcp/__init__.py` (NEW).

## Acceptance

- The MCP server starts (stdio) and answers `portfolio_stats` and
  `lookup_account` queries with real data from the analytics FeatureFrame.
- The RiskAuditorAgent, given a real (or scripted) Qwen function-calling
  exchange, cites evidence pulled via MCP — the `DisputeRound.evidence` array
  references values the tools returned.
- Tests stub the MCP client (no subprocess needed in CI); a live smoke test
  (skip without creds) exercises the real stdio round-trip.
- All existing tests green; the auditor's evidence-citation path is tested with
  a scripted tool response.

## Out of scope

- SSE transport / Responses-API native attachment (WA-021 stretch).
- Rebuttal/Arbiter (WA-016) — this wires the Skeptic's tools only.
- Live Qwen testing (needs DASHSCOPE_API_KEY).
