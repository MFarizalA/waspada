# WA-087 · Secrets hardening — RAM-role STS for OSS + KMS for real secrets (drop plaintext .env creds)

**Candidates:** Claude or backend/infra. Moves long-lived static credentials off compute; an
Alibaba-native security-posture upgrade (and a rubric/story asset).

## Problem
The FC function already has a RAM execution role granting OSS/SLS/ACR — **but the code ignores
it.** `oss.py:72` uses **static `OSS_ACCESS_KEY_ID/SECRET`** from env; `DASHSCOPE_API_KEY` /
`WASPADA_JWT_SECRET` / `DATABASE_URL` are **plaintext FC env vars** (`main.tf:390-408`). Long-lived
keys sitting on compute + visible in the console.

## Phase 1 (low-risk, do first) — OSS/SLS via the FC RAM role (STS)
- Switch `oss.py` from `oss2.Auth(static AK)` to **`oss2.StsAuth(id, secret, token)`** using the
  STS credentials FC injects (`context.credentials` / RAM-role credentials provider).
- **Remove `OSS_ACCESS_KEY_ID` / `OSS_ACCESS_KEY_SECRET`** from `.env`, `.env.example`, and the FC
  env block. Keep a **static-AK fallback** guarded by env presence so offline/tests/local dev still
  work (no STS locally).
- Confirm `fc_oss_read`/`fc_oss_write` cover the object path.

## Phase 2 — KMS Secrets Manager for the genuine secrets
- Create KMS secrets for `DASHSCOPE_API_KEY`, `WASPADA_JWT_SECRET`, the DB password (IaC).
- Fetch at cold-start via `GetSecretValue` (authorized by the FC role + a KMS-decrypt RAM policy);
  fall back to env for local/offline.
- *Interim option:* KMS-encrypted env vars (FC decrypts at runtime) — less code, weaker than
  runtime retrieval.

## Acceptance
- No `OSS_ACCESS_KEY_*` on the FC function; OSS reads/writes succeed via the RAM role on deployed FC.
- The three secrets resolve from KMS at runtime (or KMS-encrypted env); none in plaintext in the
  image/console/`.env`.
- Offline/tests unchanged (static-AK / env fallback preserved).

## Out of scope
- Rotating the actual keys (separate ops task).
- Migrating away from passlib/JWT (WA-028 lane).

## Notes
Phase 1 alone removes two static keys at low risk. Sequence Phase 2 after. Touches `oss.py` +
IaC (deploy/iac) — coordinate the IaC apply with the deploy lane.
