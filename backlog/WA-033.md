# WA-033 · Origination lane — umbrella: lift the guard + lane wiring

## Goal

Open the second decision lane: **Origination** — approve / reject / refer (and
price) new loan applications on the same risk engine and Agent Society. This is
the umbrella ticket; the module work is WA-034…WA-039. The substrate is already
lane-agnostic — this ticket removes the single hard block and adds the lane
assembly.

## What already generalizes (reuse verbatim — do NOT fork)

- Lane enum + validation: `config.py:23-25,54-61` (`LANES=(collections,origination)`,
  `WASPADA_LANE` validated). Both lanes are first-class in config already.
- `Agent` base + `ApprovalGate` (`agents/base.py`) — no lane awareness; gate
  actions are free strings.
- `AgentContext.lane` (`protocol.py:56,67`) — already carries either lane.
- **The entire debate engine + run loop** (`orchestrator.py:210-516`):
  `_resolve_disputes`, dispute memory, streaming hooks, `run()`. Score-then-contest
  works identically for an application PD.
- Arrow schema helpers `schema_from_dataclass`/`validate_table` (`schema.py:169-231`).
- OSS client signature `fetch_loans(lane=...)` (`data/oss.py:100-119`) — already
  lane-parameterized (column set is collections-only, see WA-038).

## The single hard block

`Orchestrator.plan()` raises for any non-collections lane
(`orchestrator.py:160-163`):
```python
if lane != COLLECTIONS:
    raise ValueError(f"lane={lane!r} orchestrator not implemented yet (Origination deferred).")
```

## Files / interfaces

- `waspada/agents/orchestrator.py` — remove the guard; add
  `ORIGINATION_STEP_ORDER` (same five steps: data_engineer → data_analyst →
  risk_model → risk_auditor → insight); parameterize `_build_agents()` by
  `context.lane` so each agent gets its lane-appropriate feature builder /
  policy / checks (per WA-035..WA-038). `report()` wording (`:539-565`) is
  collections-worded — make lane-aware (cosmetic).
- CLI already accepts `--lane origination` (`__main__.py:92`) — it must run
  end-to-end offline once the sub-tickets land (synthetic application snapshot).

## Acceptance

- `orch.plan("origination")` returns a step order instead of raising.
- Offline CLI run `--lane origination` completes end-to-end on a synthetic
  application snapshot with debate + gate + payload.
- Collections behavior byte-identical (regression: existing suite green).
