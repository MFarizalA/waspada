# WA-019 · AgentDialogue polish + "Run live (Qwen)" button

## Goal

The AgentDialogue panel (`dashboard/src/components/AgentDialogue.tsx`) is built
and renders the fixture. Polish it for the demo and wire a "Run live (Qwen)"
action that hits `/api/run?brain=qwen`, with a loading state so judges see the
society think. This is the highest-value demo surface — the debate itself.

## Files / interfaces

- `dashboard/src/components/AgentDialogue.tsx` — polish (already functional).
- A "Run live (Qwen)" button + loading state, wired to `POST /api/run?brain=qwen`.
  - On success: replace the fixture payload with the live payload (same frozen
    shape — the backend emits `agent_dialogue` per WA-014).
  - On error / timeout: clear messaging (don't leave the user staring at a spinner).
- `dashboard/src/App.tsx` or a small hook for the fetch + state.

## Acceptance

- The panel renders the live payload (not just the fixture) when "Run live" succeeds.
- Loading state is visible; errors surface a message.
- `tsc -b && vite build` green on the host (not just the container —
  node_modules portability lesson from the coordination skill).
- The fixture path still renders (default load) unchanged.

## Out of scope

- SSE streamed debate (WA-022 stretch) — this is a request/response button.
- Backend changes (WA-014 owns the payload shape).

## Depends on

- WA-014 for the live `agent_dialogue` payload (but polish work can proceed now
  against the fixture; the live button is inert until WA-014 + WA-018 land).
