# WA-034 · Origination data contract (application-time types)

## Goal

Define the Origination lane's frozen contract types in `waspada/schema.py` —
**additive**, beside the collections types (never mutate the frozen collections
shapes). An applicant has no payment history, so the collections contract does
not fit.

## What's collections-shaped today (schema.py)

- `RawLoans` (`schema.py:46-62`): `outstanding_principal` (L60), `total_paid`
  (L61), `current_status` (L62) are **post-disbursement** — they don't exist at
  application time. Fields that DO carry over: `loan_id`→`application_id`,
  `amount`, `term`, `rate` (as *requested/offered*, not booked), `grade`,
  `annual_income`, `dti`, `issue_date`→`application_date`, `purpose`, `region`.
- `FeatureFrame` (`schema.py:71-92`): `loan_age`, `payment_ratio`,
  `outstanding_ratio`, `delinquency_status` (L86-89) all require a seasoned
  loan. `label_default` (L91) = *eventual charge-off of an existing book*.
- `ScoredAccounts.recommended_action` (`schema.py:116`) = `call|watch|auto-cure`
  — collections-only vocabulary.
- `PortfolioHealth` (`schema.py:125-135`) is explicitly "(collections lane)":
  NPL ratio / vintage default rate / status mix.

## Files / interfaces (additive)

- `RawApplications` — application-time snapshot: `application_id`, `amount`,
  `term`, `requested_rate`, `grade`, `annual_income`, `dti`,
  `application_date`, `purpose`, `region`, `employment_length` (available in
  the LendingClub source), plus an outcome column for training
  (`funded` + `funded_default` — see label note).
- `ApplicationFeatureFrame` — application-time features + `label_default`
  (funded-then-defaulted; see honesty note below).
- `ScoredApplications` — `p_default`, `score_band` (Q1..Q5 reused), `segment`,
  `recommended_action ∈ {approve, refer, reject}`, optional `priced_rate`.
- `OriginationHealth` — approval rate, projected default rate of approved book,
  expected-loss of approved volume, band mix.
- Reuse `schema_from_dataclass`/`validate_table` (`schema.py:169-231`) verbatim
  — they work for any dataclass.

## Label honesty note (do not fake)

The LendingClub snapshot only contains **funded** loans — there is no
rejected-applications outcome data. The origination label is therefore
"funded-then-defaulted" (reject-inference is impossible from this data). State
this in the module docstring the same way the collections provenance note does
(`schema.py:20-27`); never claim approve/reject ground truth.
