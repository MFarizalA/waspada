# WA-085 · Substantiate the single-agent efficiency baseline (conformance A3)

**Candidates:** owner (live run — holds the API key) or backend (proxy design decision).
Closes gap A3 from `backlog/architecture-conformance-review.md`. **Do NOT fabricate a
number** — the current `not_run` is an integrity rail, not a bug.

## Why this matters
Track 3 explicitly requires **"a measurable efficiency gain over single-agent
baselines."** Today the head-to-head is one-sided: the society arm is measured (on a
labeled `mock-deterministic` proxy), but `AGENT_SOCIETY_BENCH.json` shows
`single_agent_baseline.status = "not_run"` (needs a live Qwen brain +
`DASHSCOPE_API_KEY`; `waspada/bench_society/run_bench.py:525-534,564-572`). The
"~10.7× fewer calls" number is honest on the **calls** axis (baseline = 1.0 call/account
by construction), but the baseline's **quality** (recall/precision at call-tier) is empty —
so the "no quality sacrifice" half of the claim is unsubstantiated. A judge who opens the
JSON sees a comparison with no measured opponent.

## Two honest paths (pick one — this is the decision)

### Path 1 (recommended) — run the live baseline
Owner runs `python -m waspada.bench_society.run_bench` with `DASHSCOPE_API_KEY` set +
`openai` installed. The baseline makes **one `qwen3.7-plus` call per test account** over a
small holdout (`run_bench.py:574-610`) — cheap (tens–hundreds of accounts, ~cents of tokens)
and gives a **real measured** recall/precision to sit beside the society arm.
- *Cost:* a few cents of Qwen tokens; a few minutes.
- *Result:* `single_agent_baseline.status="ran"` with real numbers → the claim is substantiated.

### Path 2 — a labeled deterministic proxy baseline (backend design decision)
Add a `mock-deterministic` single-agent brain (mirroring the society arm's mock brains),
clearly labeled, for an offline-reproducible comparison.
- **Risk / the reason this needs a decision:** the proxy must be a **fair** representation
  of a one-shot LLM decision. A crude heuristic would *understate* the baseline →
  *overstate* the society's advantage — the exact gaming the authors avoided with `not_run`.
  If chosen, the heuristic is the backend owner's call and must be defensible, and the
  arm must stay tagged `mock-deterministic` (never presented as a live result).

## Acceptance
- `single_agent_baseline.status == "ran"` with recall/precision + calls-per-account.
- The hero number cites a **measured** (Path 1) or **clearly-labeled-proxy** (Path 2) opponent.
- The comparison brains are documented (society vs baseline: same class of brain, or the
  asymmetry stated honestly).

## Out of scope
- Faking/fabricating the baseline (explicitly forbidden — see above).
- Changing the society arm's measured numbers.
