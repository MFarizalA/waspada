# WA-007 · CPU-vs-GPU benchmark harness

## Goal
The "why GPU" proof: a reproducible benchmark running the SAME feature+model
pipeline on pandas+sklearn (CPU) vs cuDF+cuML (GPU) over N rows, timing each
stage, reporting speedup. The headline acceleration number for the demo.
Designed around a **CPU-stressing workload** (heavy groupby + many-feature
engineering + tree model), not trivial ops (addresses the "weak speedup on
trivial ops" risk in HACKATHON.md).

## Files / interfaces
- `waspada/bench/harness.py`:
  - `run_benchmark(row_counts=[100_000, 1_000_000], stages=["features","model"])
    -> BenchReport` — times each stage on both stacks; reports wall-clock +
    speedup.
  - `BenchReport` — JSON: per-stage `{cpu_s, gpu_s, speedup_x}`, peak VRAM, row
    count, notes.
- `gpu/run_bench_gpu.py` + `cpu/run_bench_cpu.py` — the two stack runners
  invoked from the harness (GPU via WSL).
- `bench/results/` — gitignored; commit one `bench/LAST_RUN.json` sample for the
  README.

## Acceptance
- Produces a `BenchReport` for ≥2 row counts showing GPU vs CPU wall-clock per
  stage.
- Reports peak VRAM; documents any chunking at 4 GB.
- Workload is CPU-stressing (documented: many-feature groupby + tree model).
- Summary number is **honest** — no fabricated timing. If GPU barely wins, say
  so and explain why.

## Out of scope
Optimizing either stack, model accuracy (WA-005/QA), the dashboard, Freddie Mac
scale (noted as future).

## Note
Freddie Mac (tens of millions of panel rows) is the **stretch** scale-up for the
headline big-number benchmark; the MVP benchmark runs on the LendingClub
cross-sectional set (~2.2M loans × many features + tree model). Both stages
inherit the corrected cross-sectional pipeline from WA-004/005.

## Depends on
WA-004 (features), WA-005 (model) — reuses their logic on both stacks.
