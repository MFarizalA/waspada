# WA-029 · Data Engineer agent (qwen3.6-flash function-calling + dlt/DuckDB)

## Goal

Promote `ingest` from a deterministic Tier-1 step into a Tier-2 Data Engineer
agent: a qwen3.6-flash function-calling loop that decides which data-quality
checks to run on the freshly-loaded book before anyone trusts it. The existing
freshness + schema gate stays as the deterministic core — the agent adds a
reasoning layer on top, it does not replace the gate.

## Files / interfaces

- `waspada/agents/data_engineer.py` (new) — the agent + its function-calling loop.
- Quality tools registered via the same `register_tool` pattern as ingest's
  `fetch` (→ stubbable in tests): `validate_schema`, `null_rates`,
  `profile_column`, `detect_anomalies`, backed by dlt + DuckDB over the OSS
  Parquet.
- `waspada/agents/orchestrator.py` — swap `ingest` → `data_engineer` in the
  step order; keep the deterministic freshness/schema gate inside it.
- dlt load + a frozen schema contract on the OSS Parquet (reject drift).

## Acceptance

- The agent runs a real multi-hop tool loop (Qwen picks the next check from what
  it saw), not a fixed sequence — verifiable in the step log.
- Dirty/malformed data → `blocked` (gate still fails loud); an unparsable tool
  step → falls back to the default check set, never skips validation.
- Existing offline tests stay green on MockLLM (scripted tool replies); no
  network in CI.

## Out of scope

- The Data Analyst / feature building (that's WA-030).
- RDS/postgres federation (separate, WA-028 territory).
- Any change to the frozen RawLoans → FeatureFrame contract.
