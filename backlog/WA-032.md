# WA-032 · Human-configurable decision matrix (RiskPolicy)

## Goal

Let an analyst/admin define the risk **decision matrix** — the band→action
mapping and the alert thresholds — in a committed JSON policy file instead of
editing Python constants. Today every risk-policy value is hard-coded; nothing
policy-shaped flows through `waspada/config.py` (OSS + lane only).

## Where the policy lives today (the surfaces to lift)

| Surface | Location | Today |
|---|---|---|
| **Band→action matrix** | `waspada/insight/ranking.py:46-52` `ACTION_BY_BAND` (Very High→call, High/Medium→watch, Low/Very Low→auto-cure), used in `rank()` at `:92` | module const, **no kwarg** |
| Alert thresholds | `ranking.py:57-58` (`DEFAULT_NPL_THRESHOLD=0.20`, `DEFAULT_VINTAGE_THRESHOLD=0.15`) | `alerts()` takes kwargs (`:176-177`) but `InsightAgent` passes none (`insight.py:76`) |
| NPL buckets | `ranking.py:63` `_NPL_BUCKETS = {"Default","31-120","16-30"}` | bare const, used in `segment_health()` `:140` |
| Risk levels (Very Low–Very High) | `waspada/model/risk.py:262` `np.percentile(probs,[20,40,60,80])` | inline literals; **relative** quintiles (out of scope here — see stretch) |

## Design (frozen by owner review)

- **New `waspada/policy/` package**: a frozen `RiskPolicy` dataclass —
  `band_to_action: Dict[str,str]`, `npl_threshold: float`,
  `vintage_threshold: float`, `npl_buckets: frozenset` (room for `band_edges` /
  `lgd` later). `RiskPolicy.default()` returns the current constants **verbatim**
  (regression: behavior unchanged when no file). `load_policy(path|None)` reads
  **JSON** (no new dep; `None` → the committed
  `waspada/policy/default_policy.json`, the file a human edits) and **validates**:
  actions ⊆ `{call,watch,auto-cure}`, bands ⊆ `RISK_LEVELS` (Very Low..Very High), thresholds ∈ [0,1] —
  `ValueError` on drift, same fail-loud discipline as the lane check.
- **Wire-through** (defaults preserve current behavior exactly):
  - `rank()` gains `action_by_band: Optional[Dict[str,str]] = None` (fall back to
    the constant); `segment_health()` gains `npl_buckets=None`.
  - `InsightAgent.__init__(..., policy: Optional[RiskPolicy] = None)` → passes the
    matrix/thresholds/buckets into `_rank`/`_segment_health`/`_alerts`
    (`insight.py:74-76`).
  - `Orchestrator(..., policy=None)` → builds `InsightAgent(..., policy=self.policy)`
    (`orchestrator.py:201`).
  - Entry points load the file: CLI `__main__.py` and API `_build_demo_orchestrator`
    via `load_policy(os.environ.get("WASPADA_POLICY_FILE"))`. Document
    `WASPADA_POLICY_FILE=` in `.env.example`.
- **Frontend: NO change.** `ScoredAccounts.recommended_action`/`score_band` are
  bare `str` in the frozen contract (`schema.py:112,116` — enums documented, not
  enforced) but the dashboard badges **hard-depend** on the literals
  (`ActionBadge.tsx:6-8`, `types.ts:33`). `load_policy` validation keeps values
  in-vocabulary, so the UI renders matrix output unchanged.

## Acceptance

- `RiskPolicy.default()` equals the current constants; a full offline run with no
  policy file is byte-identical to today (regression test).
- `load_policy` reads the committed JSON; rejects out-of-vocab action / bad band /
  out-of-range threshold with `ValueError`.
- Editing the JSON (e.g. High→call) changes `rank()` output; custom thresholds
  change `alerts()` output — with **no code change**.
- `tests/test_policy.py` green offline; full suite stays green.

## Out of scope (stretch later)

- Absolute band edges (replace the relative-quintile `[20,40,60,80]` literals in
  `risk.py:262`) — a policy `band_edges` field slot is reserved.
- LGD / Expected-Loss config — EL is not computed in the backend yet (WA-024 is
  frontend-label-only; `grep expected_loss **/*.py` → 0 hits).
- A dashboard settings UI (would need a persist store + save/load endpoint).
- Frontend severity bands (`dashboard/src/lib/severity.ts:35-39`) — frontend-only
  today; fold into the policy when a config channel to the SPA exists.
