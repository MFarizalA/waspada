# WA-024 · Expected Loss (PD × LGD × EAD) (~2h)

## Goal

The most credit-credible number we can show: per-account and portfolio
**Expected Loss = PD × LGD × EAD**, with assumptions labeled on-screen. Turns
the work-list from "risky accounts" into "rupiah at risk" — how a risk committee
actually ranks work.

## The formula + assumptions (label these in the UI)

- **PD** = the model's `p_default`.
- **LGD** = 45% flat (Basel foundation-IRB benchmark for unsecured consumer
  credit — labeled as an assumption, not measured).
- **EAD** = `outstanding_principal` (defensible for amortizing installment loans
  — no revolving/undrawn component; labeled).

## Files / interfaces

- Backend (Bimo): `expected_loss = p_default × 0.45 × outstanding_principal` into
  the payload (additive optional key on work-list rows); portfolio EL sum into
  `portfolio_health`.
- Frontend (Kirana): display per-account EL in the work-list + portfolio EL in
  the health panel; assumptions labeled (a small "assumptions" footnote or tooltip).

## Acceptance

- Work-list rows carry `expected_loss` (additive; older payloads without it stay valid).
- Portfolio health carries total expected loss.
- UI labels LGD=45% and EAD=outstanding_principal as assumptions visibly.
- No fake metrics: vintage default rate by cohort and status mix are computable
  (already shipped); cure rates and true band migration are NOT computable from
  one snapshot — do not fake them (HACKATHON.md honesty rails).
- All existing tests green.

## Out of scope

- EL-weighted product×region heat-map + HHI concentration (stretch, fold in only
  if this lands early).
- LGD modeled per-account (flat 45% is the labeled assumption).

## Notes for the worker

- ~2h scope. Additive optional keys only — don't break the frozen payload contract.
- The assumptions label is the credibility feature; don't hide it.
