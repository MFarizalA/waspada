# WA-072 · Branding — repo + product presentation pass

**Candidates:** Kirana if revived (frontend craft), otherwise Claude or owner.
Small surface, high judge-visibility.

## Goal

Make the repo and product look like one coherent, deliberately-named thing
when a judge opens it. First-impression polish, not redesign.

## What to do

1. **Repo About section** (GitHub settings — owner-console or owner): description
   one-liner, topics/tags (`qwen`, `alibaba-cloud`, `agent-society`,
   `hackathon`, `credit-risk`), website = the live URL. The MIT license must
   be **detected by GitHub** (shows in the About sidebar) — requires a
   standard-format `LICENSE` file at root; verify detection works once the
   repo is public.
2. **Naming consistency.** WASPADA everywhere — repo name, README title,
   dashboard `<title>` + header, HACKATHON.md, video script (WA-074/075).
   Grep for stray old names/inconsistencies. Agent display names use the
   industry terms (Risk Auditor / Actuary / Credit Arbiter), never internal
   codenames.
3. **README hero.** Top of README: one-paragraph what/why, the live URL, the
   3-min video link (placeholder until WA-074 lands), track badge (Track 3:
   Agent Society). Judges decide in the first screen whether to read on.
4. **Dashboard shell.** Title tag, favicon (any simple mark is fine), header
   label — whatever renders in the demo video should look intentional.

## Acceptance

- GitHub repo page: description + topics + website + detected license all
  visible (screenshot or owner confirmation).
- `grep -ri` finds no stale product names; agent industry names used in all
  judge-facing surfaces.
- README first screen contains: what it is, live URL, video link, track.
- Dashboard title/favicon/header committed.

## Out of scope

- Logo design, marketing site, any visual redesign of the dashboard.
