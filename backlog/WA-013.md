# WA-013 · Origination lane (DEFERRED — do not dispatch)

## Goal
The additive second lane: origination risk scoring on new applications using
LendingClub application attributes. Approve / reject / limit / price. Reuses the
shared engine (ingest, GPU pipeline, ranking, dashboard, agents) with
origination-specific features/label/recommendation.

## Files / interfaces (planned, not built)
- `waspada/features/origination.py` — features: income, DTI, tenure, amount,
  collateral, employment.
- Label: defaulted over loan life (from LC final status).
- Recommendation: approve / reject / limit / price band.
- `waspada/agents/` — origination variants of pipeline agents + orchestrator
  lane switch (`--lane origination`).

## Acceptance (when built)
- `python -m waspada.agents --lane origination` produces an origination decision
  payload.
- Dashboard shows an origination view alongside Collections.
- Benchmark includes the origination pipeline.

## Out of scope (for now)
Everything — this is a placeholder to reserve the lane. **Do not dispatch until
the Collections lane ships and the owner says go.**

## Rationale
Collections/EWS is the hero (sharpest bottleneck, biggest data → best
acceleration number). Origination is additive on the same engine and comes
second, per HACKATHON.md sequencing.
