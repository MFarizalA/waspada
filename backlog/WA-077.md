# WA-077 · Cut the API over to real OSS data — remove the synthetic stub from prod

**Owner directive (2026-07-19):** "not production ready" — the demo-mode
synthetic stub must go. The API reads real data from OSS or it fails loudly.
No silent fallback to synthetic in any API path.

## Current state (the problem)

`api/main.py::_build_demo_orchestrator` force-registers a stub fetch over
`_sample_raw_table(n=200)` on the DataEngineerAgent — overriding the real
OSS read path that already exists (`waspada/data/oss.py::fetch_loans`,
tested, WA-047-honest). Both `/api/run` and `/api/run/stream` use this
builder. The deployed product serves synthetic data while the repo claims
a real OSS lakehouse.

## Goal

The API's orchestrator fetches via `oss.fetch_loans()` against the real
Raw/Staging/Mart surface. Synthetic `_sample_raw_table` is REMOVED from all
API paths (it may stay in the CLI offline path `waspada/agents/__main__.py`
— that's a dev tool, not the product).

## Files / interfaces

- `api/main.py` — `_build_demo_orchestrator`: stop registering `_stub`;
  the DataEngineerAgent's `fetch` tool must resolve to `oss.fetch_loans`
  (check what the agent's DEFAULT fetch tool is — if the agent already
  defaults to `oss.fetch_loans` when no stub is registered, this is a
  deletion, not new code). Rename the builder if "demo" in the name offends
  the new posture — your call, keep the diff small.
- `api/main.py` lifespan/startup — add a **startup probe**: on boot, verify
  OSS reachability (head_object on the configured key). If OSS is
  unreachable or the object is missing, the app must STILL BOOT (health
  stays ok) but `/api/run` returns a clear 503 `{"detail": "data source
  unavailable: <reason>"}` — never silent synthetic.
- `deploy/iac/main.tf` — FC function `environment_variables`: add
  `OSS_RAW_BUCKET`, `OSS_ENDPOINT` (internal), `OSS_ACCESS_KEY_ID`,
  `OSS_ACCESS_KEY_SECRET` (from the existing secrets.tfvars values — check
  `variables.tf` for what's already declared; DO NOT hardcode secrets in
  main.tf, wire from var). Also `OSS_KEY` if the config uses a non-default.
- `.env.example` — document the OSS vars the API now requires.

## Behavior contract

| Situation | /api/run behavior |
|---|---|
| OSS reachable, real data present | runs on real data, payload `meta.source` says so |
| OSS unreachable / no credentials | 503 with clear detail, app stays up |
| brain=mock | still allowed (mock = the LLM brain, NOT the data — data is real either way) |

`brain=mock` vs `brain=qwen` selects the reasoning LLM. Data source is NOT
selected by anything — it is always OSS. Kill any confusion on this in
comments.
