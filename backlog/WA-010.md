# WA-010 ¬∑ Orchestrator agent (the primary agent)

## Goal
The primary agent ‚Äî plans the Collections run, coordinates the four pipeline
agents in order, holds the **human approval gate** before the work-list is
released, and reports to the analyst in plain language. The agent the demo leads
with.

## Files / interfaces
- `waspada/agents/orchestrator.py`:
  - `Orchestrator(Agent)` ‚Äî `plan(lane="collections")` builds the step
    sequence; `run()` executes ingest‚Üíanalytics‚Üírisk-model‚Üíinsight, passing
    artifacts via `AgentContext`, calling `ApprovalGate` before publishing the
    work-list.
  - `report(payload) -> str` ‚Äî plain-language analyst summary (top risks,
    portfolio health, alert count).
- `waspada/agents/__main__.py` ‚Äî CLI: `python -m waspada.agents --lane
  collections` runs the full orchestrated pipeline, prints the report, writes
  the dashboard payload.
- `tests/test_orchestrator.py` ‚Äî full run with mock LLM + stubbed data; asserts
  step order, gate invocation, payload emitted, report non-empty.

## Acceptance
- `python -m waspada.agents --lane collections` produces the `DashboardPayload`
  + an analyst-readable report.
- Approval gate invoked before the work-list is released; rejection
  short-circuits with a clear message.
- Step ordering logged; a failure in any stage surfaced (not swallowed).
- End-to-end works offline (mock LLM, small/stubbed data).

## Out of scope
Dashboard rendering (WA-011), Origination orchestration (deferred), QA (WA-012).

## Depends on
WA-009 (pipeline agents), WA-008 (framework + gate).
