# WA-022 · SSE-streamed live debate (STRETCH)

## Goal

Stream the debate live in the UI — judges watch the Skeptic challenge, the
Actuary defend, the Judge rule, in real time. The highest-impact demo flourish.

## Delivered

- `GET /api/run/stream` returns `text/event-stream` SSE frames.
- Auth accepts `?token=` query param (EventSource-friendly) via `current_user_ws`
  while `POST /api/run` remains Bearer-header only.
- Event shapes match `dashboard/src/lib/useLiveDebateStream.ts`:
  - `round` — round_no, speaker, model, claim, confidence, evidence
  - `resolution` — loan_id, resolution, resolved_by, rationale
  - `done` — terminal stream marker
- Orchestrator debate hooks (`on_round_complete`, `on_dispute_resolved`) emit
  events from the worker thread onto an `asyncio.Queue`.
- Full test suite: `292 passed, 7 skipped, 1 xfailed`.

## Out of scope until

Core lands first. Depends on WA-019 (the Run live button) + a streaming backend
endpoint (SSE on `/api/run?brain=qwen`).
