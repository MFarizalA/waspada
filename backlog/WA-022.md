# WA-022 · SSE-streamed live debate — backend endpoint

## Goal

Stream the debate live in the UI — judges watch the Skeptic challenge, the
Actuary defend, the Judge rule, in real time. The highest-impact demo flourish.

**Frontend already shipped** (commit `654d044`, Kirana):
`dashboard/src/lib/useLiveDebateStream.ts` opens an `EventSource` on
`/api/run/stream?brain=qwen` and renders rounds as they arrive, with graceful
fallback to the request/response "Run live" button. **The remaining work is the
backend endpoint** — hence owner flipped to backend.

## KEY FACT (verified — changes the scope)

The orchestrator streaming hooks are **already built and wired**. Do NOT touch
the orchestrator debate loop.

- Constructor params `on_round_complete: Callable[[Dispute, DisputeRound], None]`
  and `on_dispute_resolved: Callable[[Dispute], None]` (orchestrator.py:84–85).
- They fire at every correct point in `_resolve_disputes` (lines 342, 357, 367,
  382, 390, 398, 451, 452, 510) via `_emit_round`/`_emit_resolved`, which already
  swallow callback exceptions so a stream hook can never fail the run.

The job is **only the FastAPI endpoint** that wires those callbacks to an SSE
response. Nothing in `waspada/agents/` needs editing.

## The frozen event contract (frontend is already built to it)

Match these SSE `data:` frames byte-for-byte (`useLiveDebateStream.ts`):

```
data: {"type":"round","round_no":1,"speaker":"risk_auditor","model":"qwen3.6-flash","claim":"...","confidence":0.72,"evidence":["..."]}

data: {"type":"resolution","loan_id":"LN00961668","resolution":"upheld","resolved_by":"arbiter","rationale":"..."}

data: {"type":"done"}
```

- SSE framing: each event is `data: <json>\n\n`; `media_type="text/event-stream"`.
- `resolution` ∈ `upheld | overridden | escalated_approved | escalated_rejected`;
  `resolved_by` ∈ `risk_model | arbiter | human`. Read them off the resolved
  `Dispute` — reuse `Dispute.to_dict()` (protocol.py); don't hand-roll.
- Exactly one `resolution` per dispute (from `on_dispute_resolved`); one terminal
  `done` after `orch.run()` returns.

## Architecture decisions (fixed)

1. **Orchestrator is sync; endpoint is async.** Run `orch.run(ctx)` in a worker
   thread (`asyncio.to_thread`). The callbacks (called from that thread) push
   JSON dicts onto an `asyncio.Queue` via `loop.call_soon_threadsafe(...)`. The
   async generator drains the queue and yields SSE frames; when the run thread
   finishes, enqueue a `done` sentinel. Never block the event loop; never touch
   request/FastAPI objects from the callbacks — only enqueue plain dicts.
2. **Reuse the `/api/run` setup verbatim** — same synthetic snapshot, same
   `DataEngineerAgent` fetch-stub, same auto-approve gate, same `brain` param.
   Only difference: pass `on_round_complete`/`on_dispute_resolved` into
   `Orchestrator(...)`. Factor the shared build into a helper so `/api/run` and
   `/api/run/stream` don't drift.
3. **Auth via `?token=`** (see Files).
4. **`brain=mock` must still produce a valid (possibly empty) stream** for tests
   — no disputes → stream is just `done`. A *visible* debate needs `brain=qwen`
   or a scripted mock. Note this in the docstring.

## Sketch

```python
@app.get("/api/run/stream")
async def run_stream(brain: str = "mock", _user: dict = Depends(current_user_ws)):
    q: asyncio.Queue = asyncio.Queue()
    loop = asyncio.get_running_loop()
    def on_round(d, r):    loop.call_soon_threadsafe(q.put_nowait, _round_event(d, r))
    def on_resolved(d):    loop.call_soon_threadsafe(q.put_nowait, _resolution_event(d))
    orch = _build_demo_orchestrator(brain, on_round_complete=on_round,
                                    on_dispute_resolved=on_resolved)
    async def runner():
        await asyncio.to_thread(_run_demo, orch)
        await q.put({"type": "done"})
    async def event_gen():
        task = asyncio.create_task(runner())
        try:
            while True:
                ev = await q.get()
                yield f"data: {json.dumps(ev)}\n\n"
                if ev.get("type") == "done":
                    break
        finally:
            task.cancel()
    return StreamingResponse(event_gen(), media_type="text/event-stream",
                             headers={"Cache-Control": "no-cache",
                                      "X-Accel-Buffering": "no"})
```

`_round_event(d, r)` maps a `DisputeRound` → the `round` frame; `_resolution_event(d)`
maps the resolved `Dispute` → the `resolution` frame. Field names from
`Dispute.to_dict()`.

## Gotchas

- Don't `json.dumps` a `Dispute` dataclass directly — map explicitly / via
  `to_dict()` so the frame matches the frontend contract.
- SSE needs the blank line (`\n\n`) after each `data:` or the browser won't
  dispatch the event.
- The run thread can't be truly cancelled on client disconnect — that's fine;
  just stop yielding (the `finally: task.cancel()` handles the generator side).
