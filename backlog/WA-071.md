# WA-071 · Sanity test — the "judge clones the repo" sweep

**Candidates:** Bimo or Claude. This is the final functional gate before submission.

## Goal

Simulate exactly what a judge does: clone the public repo fresh, follow the
README verbatim, and confirm everything works. Plus a live-prod smoke. Any
friction a judge would hit, we hit first.

## What to test

1. **Fresh-clone quickstart.** In a clean directory/machine profile: clone,
   follow README Quick start **verbatim** (venv, install, env setup, run
   tests, run the demo pipeline). Every command must work as written — note
   any missing step, stale command, or undocumented dependency. This is the
   highest-value check: a broken quickstart is a dead submission.
2. **Full test suite.** `pytest -q` from the fresh clone: expect
   425 passed, 7 skipped, 1 xfailed. Skips must be the documented live-only
   ones, not new breakage.
3. **Live prod smoke.** Against
   `https://waspadaprod-api-vouqzqqkiu.ap-southeast-1.fcapp.run`:
   `/api/health` → ok; seeded judge login (analyst@waspada.demo / waspada123)
   → token; `POST /api/run?brain=mock` → 200 with populated payload.
   Read-only posture; no prod data modification.
4. **Dashboard local run.** `cd dashboard && npm ci && npm run build &&
   npm run dev` from the fresh clone — builds clean, renders, login works
   against prod API (CORS already verified).
5. **Repo hygiene.** No secrets anywhere (`git grep` for key patterns,
   check `.env` is gitignored, `.env.example` has no real values). LICENSE
   file present at root (MIT per README — verify the file actually exists).

## Acceptance

`backlog/WA-071-sanity.md`: per-item pass/fail with evidence (commands +
output), every failure rated (judge-blocking / cosmetic) with a suggested
fix owner. No fixes in this ticket — findings only.
