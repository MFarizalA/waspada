# WA-041 · Native tools/tool_calls function calling

## Goal

Replace JSON-mode prompt-and-parse tool selection with Qwen's native
`tools` / `tool_calls` API. This is the biggest HACKATHON.md 🟡 — it sits on the
"Technical Depth & Engineering (sophisticated QwenCloud API use)" rubric axis,
which is live while judging is ongoing. Turning 🟡 → ✅ is direct rubric value.

Today the Data Engineer, Data Analyst, and Risk Auditor agents use JSON-mode
text (`_parse_tool_call`, `_parse_view_json`) to decide which tool to invoke.
Qwen decides *which* tool by emitting a JSON blob we parse, then we execute
the tool in Python. This works but is not "sophisticated QwenCloud API use" —
it's prompt engineering with a JSON safety net.

Native function calling means: we declare tool schemas via the `tools` parameter,
Qwen emits `tool_calls` in its response, and the loop continues until Qwen stops
calling tools. *Qwen decides* when to pull context, not hard-wired Python.

## What changes

- `waspada/agents/llm.py` (`QwenLLM.complete()` ~line 203) — add `tools`
  parameter support. The OpenAI-compatible DashScope endpoint accepts:
  ```python
  tools=[{"type": "function", "function": {"name": ..., "description": ..., "parameters": {...}}}]
  ```
  and returns `response.choices[0].message.tool_calls` (list of
  `{id, function: {name, arguments}}`).

- `waspada/agents/risk_auditor.py` — the `run()` method's LLM call switches
  from JSON-mode (`response_format={"type": "json_object"}`) to the `tools`
  parameter. The `portfolio_stats` and `lookup_account` tools become native
  function schemas, not JSON-mode prompt instructions.

- `waspada/agents/data_analyst.py` — the function-calling loop (`_tool_query`,
  `_tool_correlation`, etc.) becomes native tool schemas. The hop budget
  (`DEFAULT_QUERY_BUDGET`) stays — it bounds the loop regardless of mechanism.

- **MockLLM** (`llm.py`) — add `tool_calls` support to the mock so offline
  tests exercise the native path. The scripted-brain pattern (list of
  pre-set responses) should support both `content` and `tool_calls` shapes.
