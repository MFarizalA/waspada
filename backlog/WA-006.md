# WA-006 · Ranking, segmentation & work-list

## Goal
Turn `ScoredAccounts` into the decision-support output: a ranked collections
work-list + portfolio-health aggregates the dashboard needs, plus segment
deterioration alerts. Emits the `DashboardPayload` (frozen contract) — the
hand-off to the frontend.

## Files / interfaces
- `waspada/insight/ranking.py`:
  - `rank(scored, top_n=…) -> WorkList` — sort by `p_default` desc; attach
    `recommended_action` ("call" / "watch" / "auto-cure") by band.
  - `segment(scored) -> PortfolioHealth` — groupby product/region → computable
    **cross-sectional** aggregates: `npl_ratio` (% in delinquent/default
    status), `vintage_default_rate` (default rate by `issue_date` cohort),
    `status_mix` (distribution of current statuses by segment). NB: **true roll
    rates need a monthly panel (Freddie Mac, stretch)** — not computed here.
  - `alerts(health) -> [Alert]` — flag cohorts whose `npl_ratio` /
    `vintage_default_rate` exceeds a configurable threshold vs the portfolio
    baseline.
  - `to_dashboard_payload(work_list, health, alerts) -> DashboardPayload`
    (JSON-serializable, matches contract).
- `tests/test_ranking.py` — synthetic scored set; asserts top-N order, segment
  math (NPL ratio + vintage default rate), alert fires on a deteriorating cohort.

## Acceptance
- `to_dashboard_payload()` returns JSON matching `DashboardPayload` (validated
  against `waspada.schema`).
- Ranking deterministic; segment aggregates match a hand-computed expected on
  the toy set.
- At least one alert type with a configurable threshold.
- Pure CPU (pandas/Arrow) fine here — no GPU required.

## Out of scope
Dashboard UI (WA-011), model scoring (WA-005), benchmark (WA-007), true roll
rates (Freddie Mac panel stretch).

## Depends on
WA-001 (`DashboardPayload`, `ScoredAccounts`), WA-005 (scored accounts).
